写在前面：这套方法是我帮一个朋友做了一份汇报演示之后，她说"这个我自己也想做"，于是我把过程整理成了这篇文章。她不会写代码，电脑里只有 Office、WPS 和 Edge 浏览器，最后她用我教的提示词，半小时独立做出了第二份。

如果你也是"会用电脑、不会写代码、又时不时要做 PPT"的那种人，这篇就是写给你看的。

## 一、先看场合，别盲跟

这种 PPT 长什么样？就是你偶尔在 B 站、公众号上看到的"网页版 PPT"——能放动画、能交互、配色比 PowerPoint 模板高级一截，全屏一打开就有那种"这家伙是不是请人做的"的效果。

但请先认清边界。**它适合：**

- 自己上去讲的会议、培训、分享会、述职、节庆主题展示
- 用电脑或会议室投影仪，按方向键翻页
- 你想让效果"亮一下"的场合

**它不适合：**

- 发给同事 / 领导用 WPS 或 PowerPoint 改
- 打印出来归档发文
- 上传到必须 `.pptx` 格式的内部系统

如果你的 PPT 最终命运是"发给别人改"，老老实实用 PowerPoint 套模板，AI 只用来润色文字。这篇就别看了。

## 二、三步上手最简版

### 第 1 步：打开 DeepSeek

浏览器进 [chat.deepseek.com](https://chat.deepseek.com)，注册登录。免费、不用翻墙、不限速。

如果你的电脑装不了客户端、上得了网就行。微信扫码登录最快。

### 第 2 步：把下面这段提示词改两个字发给它

```
你是一位资深 PPT 设计师。请用一个完整的 HTML 文件帮我做一份汇报演示，主题是：{在这里填你的主题}。

要求：
- 单文件 HTML，所有 CSS 和 JS 都内联，不依赖任何外部资源
- 8 张幻灯片，每张占满全屏，按方向键翻页
- 风格庄重大气，配色稳重，字体用衬线（宋体/思源宋体）
- 包含：封面 / 背景介绍 / 关键数据可视化 / 时间轴 / 引用名言 / 收尾
- 每页要有动画进场效果
- 在右下角加翻页按钮和当前页码
- 关键数字用大字号突出
```

把 `{在这里填你的主题}` 换成你要做的题，比如"2026 年第一季度产品复盘"或者"读书会分享：原子习惯"。

### 第 3 步：把 AI 给的代码变成可以双击打开的文件

这一步最容易卡住，看仔细：

**Windows 用户：**

1. 桌面右键 → 新建 → 文本文档
2. 双击打开记事本
3. 把 AI 给的整段代码（从 `<!DOCTYPE html>` 到 `</html>`）全部复制粘贴进去
4. 文件 → 另存为
5. 文件名：`汇报.html`（注意：是 `.html` 不是 `.txt`）
6. 保存类型选"所有文件"，编码选 **UTF-8**
7. 双击这个文件，浏览器会自动打开

**Mac 用户：**

1. 打开"文本编辑"
2. 格式 → 转换为纯文本（`Shift + Cmd + T`）
3. 粘贴代码
4. 存储为 `汇报.html`
5. 弹窗提示后缀时选"使用 .html"

打开之后看到的就是你的 PPT，按方向键翻页。

## 三、5 套抄就能用的提示词

把上面"第 2 步"的主题部分，换成下面任意一个块。其他不要动。

### 工作汇报 / 项目复盘

```
{XX 项目 / XX 部门 2026 年第一季度工作汇报}

要求同上，另外加：
- 第二页"主要指标"放 4 个关键数据，用大字号
- 中段放时间轴展示季度内的重要节点
- 倒数第二页"存在问题"3 条
- 末页"下阶段重点"3 条
```

### 学习分享 / 读书会

```
{书名 / 主题}：分享会演示

要求同上，另外加：
- 第二页"为什么读这本书"
- 中段每张幻灯片一个核心观点，配一句精华引用
- 倒数第二页"对我的启发"3 条
- 末页"推荐给谁读"
```

### 知识科普 / 培训分享

```
{某专业主题}入门讲解

要求同上，另外加：
- 第二页"为什么要了解这个"
- 中段每张幻灯片一个核心概念，配示意图（用 SVG 画）
- 倒数第二页"常见误区"
- 末页"3 条记住的话"
```

### 节庆纪念 / 周年主题

```
纪念{节日 / 历史事件}{XX} 周年

要求同上，另外加：
- 封面用最具代表性的视觉元素
- 第二页放一组震撼的数字
- 中段时间轴展示历史进程
- 末页用诗词或名言收尾
```

### 产品发布 / 路演

```
{产品名}产品介绍

要求同上，另外加：
- 第二页"用户痛点"
- 中段"我们的解决方案"配示意图
- 倒数第二页"已有数据"用大字号关键指标
- 末页"下一步路线图"
```

## 四、不满意？这样让 AI 局部改

新手最常犯的错误：每次不满意都让 AI 重写整个，然后越改越烂。

**正确姿势是局部改。** 把这些指令直接发给它：

- "把第 3 页标题改成 XXX，其他不动"
- "整体配色换成藏青 + 银白，更适合科技类"
- "在第 5 页之后插入一页，讲 XXX"
- "把第 6 页的字体放大 20%，标题加粗"
- "封面太花了，简化一下，只保留主题和年份"
- "第 4 页的时间轴改成竖向"
- "末页加上一句古诗词，要 14 字以内"

每次只改一处，AI 改完会重新发整段代码，你重新保存覆盖原文件即可。

## 五、投屏演示

会议室那台老电脑通常装的是 Edge 或 Chrome，都行。

1. 把 `.html` 文件拷到 U 盘，插会议室电脑
2. 双击打开
3. 按 **F11** 全屏
4. 方向键翻页，**Esc** 退出

提前 10 分钟到会议室连一遍投影仪，确认能正常播放。**别在大家都到齐之后才发现打不开。**

## 六、5 条红线

这 5 条比上面的技巧都重要。AI 只是工具，最后挨问的是你。

**1. 引用 — 凡引用必查原文**

AI 会幻觉。它可能给你编一段听起来很有道理、但根本不存在的"某某讲话"或"某某书里说"。任何引用、名言、数据出处，都要去原文核对，一字不差。

**2. 数据 — 凡数字必复核**

AI 编数字是常事。它给你的数字看起来都很合理，但其中可能有 1 到 2 个是它编的。任何数字都要去权威来源核（官方公报、行业报告、原始论文）。

**3. 隐私 / 涉敏材料 — 别往 AI 里发**

公司内部文件、客户数据、未公开材料、合同、个人隐私信息，**一律不要发给任何 AI 工具**，包括 DeepSeek、豆包、Kimi、ChatGPT。这条没有例外。

只用 AI 做"通用框架 + 公开信息"，敏感内容你自己手动填到 HTML 里。

**4. 人名 / 单位名 — 让 AI 写就大概率错**

AI 不知道你公司全称、同事的具体职务、领导的内部称呼习惯。这些**必须留占位**让你自己填。提示词里直接写"姓名职务请用 [XXX] 占位"。

**5. 专业术语 — 容易混淆的词列表**

凡是你领域里"差一个字意思就变"的术语，自己核一遍，不要信 AI。AI 把"召回率"和"准确率"写反、把"OKR"和"KPI"混用，是常见操作。

## 七、进阶：要做出更高水准

把这段当作"长款"提示词用：

```
你是一位顶级 PPT 设计师，需要做出能在重要场合"惊艳全场"的演示。

主题：{你的主题}

设计要求：
- 单 HTML 文件，全内联
- 8 张幻灯片，左右键翻页，F 全屏
- 配色采用一组沉稳大气的方案（自行选择主色 + 强调色 + 背景色），字体用 Noto Serif SC 衬线宋体配 Noto Sans SC 无衬线
- 每张幻灯片要有：纸纹背景纹理、章节序号水印、装饰性的角标元素
- 数字要做递增动画、关键节点要做时间轴、地理或流程信息可以用 SVG 画
- 引用区用左竖线 + 浅底块
- 加一个顶部进度条和右侧圆点缩略导航

内容结构：
1. 封面（主题 + 年份 + 副标题）
2. 主题背景与意义
3. 关键数字 6 个（用卡片网格）
4. 发展脉络（时间轴形式）
5. 核心要点（4 宫格卡片）
6. 关键人物 / 案例 / 数据（带引文或图示）
7. 路线图 / 流程图（用 SVG）
8. 收尾（金句 + 行动号召）

请按这个规范输出完整代码。
```

这一段提示词背后藏的设计语言，是我自己琢磨了好几遍才稳定下来的。直接抄，比你自己描述快 10 倍。

## 八、最后说一句

AI 是杠杆，不是替代。

它能在 30 分钟内帮你把"码字 + 排版"的体力活做完，但**内容准不准、引用真不真、措辞分寸合不合适，永远是你自己的判断力**。

做完之后，自己从头到尾过一遍，过两遍，再交出去。

这是基本功，AI 替不了。
