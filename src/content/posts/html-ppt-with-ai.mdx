周日 23:47。

领导丢来一份 word，周一早上要汇报。

你打开 PPT，模板从 2018 年用到今年。光选第一张封面就花了 12 分钟，怎么看都像 Word 截屏。

——这篇文章的目标是，**让你下次再遇到这种事，反而觉得是个机会**。不是给你 5 个套路模板，是教你一套真正能迁移的思路。读完之后，你能用同一套方法做汇报、做简历、做读书会演示、甚至做一个能投屏给小孩看的故事书。

不需要写代码。但需要你愿意花 20 分钟读完，再花 30 秒亲手试一次。

## 一、先尝甜头：30 秒跑出你的第一张 PPT

在讲任何原理之前，先动手。

打开 [chat.deepseek.com](https://chat.deepseek.com)，注册登录，免费、不用翻墙、不限速。微信扫码最快。

把下面这 80 字提示词复制进去，**只改中括号里的两个字**，按发送：

```
帮我用一个完整的 HTML 文件做一份 8 张幻灯片的汇报，主题是：[Q1 项目复盘]。
单文件 HTML，所有 CSS 和 JS 内联。
风格庄重大气，配色稳重。
按方向键翻页。
```

47 秒后，你会拿到一段从 `<!DOCTYPE html>` 开头到 `</html>` 结尾的代码。

把它**整段**复制下来：

- **Windows**：桌面右键 → 新建 → 文本文档 → 双击打开 → 粘贴 → 文件 → 另存为，文件名 `汇报.html`，保存类型选"所有文件"，编码选 **UTF-8**
- **Mac**：打开"文本编辑" → `Shift + Cmd + T` 转换为纯文本 → 粘贴 → 存储为 `汇报.html`，弹窗选"使用 .html"

双击这个文件。浏览器打开。按右方向键翻页。

够了。这就是你的第一张 PPT。

接下来这篇文章，是教你怎么把这 47 秒，升级成一份**散会以后还有人主动来问"这个怎么做的"**的汇报。

## 二、三个反直觉事实：这条路为什么走得通

刚才那 47 秒不是魔法。它能成，靠的是三件事，每一件都跟你直觉不一样。

**1. 你不是在做 PPT，你是在让 AI 写代码，代码恰好是 PPT。**

PowerPoint / WPS 是"图形软件"，你点一下、拖一下、改一下。AI 不擅长图形操作，AI 擅长写文本——而 HTML 恰好是一种文本。所以你绕开了 PPT 这个赛道，跑到 AI 的主场上较量。这是这套方法的根。

**2. 单文件 HTML 像一份完整菜谱，不是半成品。**

你拿到的代码是一个独立的 `.html` 文件，里面**所有东西都自带**——配色、字体、动画、布局。它不依赖任何外部资源，没有外网就不能用、没有插件就崩溃这种事不会发生。U 盘拷过去就能放，会议室那台老电脑也能跑。

**3. AI 不是设计师，它是个会写代码的实习生。**

这句话是这篇文章里最值得你记住的一句。

设计师懂"汇报场合的庄重感"、懂"领导喜欢哪种字号"、懂"什么颜色拿出去不出错"。**AI 不懂这些**。AI 只懂代码语法，懂上百万份网页是怎么写出来的。所以你跟它说话的方式，不能像跟设计师说话——"帮我做个好看的"它听不懂；要像给实习生写 brief——**背景 / 目标 / 风格 / 约束 / 输出格式**，五件事说清楚，它能交出极好的活。

你刚才那 80 字提示词里就藏了这五件事，你回去看一眼。

理解了这三件事，下面就分级讲，怎么把"能跑通"的 PPT，一级一级升到"让人记住"的水准。

## 三、第一级 · 能跑通的 PPT

这一级的目标只有一个：**双击能打开，投影仪能放，不在领导面前出 bug**。

听起来低，但 90% 的人第一次就是卡在这里。

### 案例引入：Q1 项目复盘

接下来全文，我用同一个案例做演示——**"Q1 项目复盘"**。无论你做什么主题（学习心得、读书会、产品发布），方法都一样，把这个案例换成你的就行。

第一级的提示词，就是上面那 80 字。我们先看看它能交出什么样的东西，再讨论这一级会踩的坑。

### 这一级最常踩的三个坑

**坑 1：代码被 ` ``` ` 包住，整段拷过去打不开**

AI 经常会把代码用三个反引号包起来——这是 markdown 写法，给程序员看的，不是 HTML 的一部分。

如果你看到 AI 的回答开头是 ` ```html ` 或 ` ``` `，结尾也是 ` ``` `——**这两行不要复制进去**。只复制中间从 `<!DOCTYPE html>` 到 `</html>` 的部分。

判断方法很简单：第一行必须是 `<!DOCTYPE html>`，最后一行必须是 `</html>`。多一个字少一个字都打不开。

**坑 2：保存的时候选错了编码，中文全是问号**

记事本默认编码不是 UTF-8。如果你保存的时候没有手动改成 UTF-8，打开 .html 之后中文会全部变成 `??????` 或者 `锟斤拷锟斤拷`。

每一次另存为，**眼睛盯着编码下拉框**，确认是 `UTF-8` 再点保存。Mac 用户用"文本编辑"默认就是 UTF-8，但**一定要先转纯文本**（`Shift + Cmd + T`），否则会保存成 RTF 富文本格式，浏览器一样打不开。

**坑 3：AI 中途断了，代码不完整**

DeepSeek 有时候输出长代码会中途停下来，留你一个不完整的 HTML。判断方法：拉到最底部，**看最后一行是不是 `</html>`**。

如果不是，发一句 `继续` 给它，它会接着写。**不要让它从头重写**——重写出来的版本跟前半段对不上，更乱。

### 红线（嵌在这一级，因为你只会在这里踩到）

**敏感材料别往 AI 里发。**

公司内部文件、客户数据、未公开报告、领导讲话稿、合同文本——这些**永远不要发给任何 AI**，包括 DeepSeek、豆包、Kimi、ChatGPT，没有例外。

正确做法：让 AI 做"通用框架 + 公开信息"，敏感内容你自己手动填到生成好的 HTML 里。比如提示词里只写"Q1 项目复盘"，不写具体项目代号；具体数据等 HTML 生成完，自己用记事本打开把 `[Q1 营收]` 改成真实数字。

**这一级走通了，你已经领先 80% 的人。**

## 四、第二级 · 不丢人的 PPT

能跑通只是基础。第二级的目标是——**放出来不被同事问"这是外包做的吗"**。

具体说，就是配色不土、字号不乱、关键数据一眼能看见。

### 在 Q1 复盘上加一层：让它有"专业感"

把第一级的提示词扩展成下面这个，发给 DeepSeek。注意——我们在**同一个对话**里继续，不开新对话：

```
不错。现在帮我加几个改动：
1. 配色用 #1a3a5c 深蓝 + #c9a961 暖金 + #f5f0e6 米白，不要用红色
2. 字体用衬线（思源宋体或宋体），标题加粗，正文常规
3. 第二页"Q1 关键指标"放 4 个数据卡片，数字用 64px 大字号
4. 第三页改成时间轴布局，展示季度内的关键里程碑
5. 第七页"存在问题"用左红线引用框
6. 末页用一句振奋的金句收尾
```

注意我说的方式——**每条都很具体**。不是"配色好看一点"，是"具体哪三个十六进制色号"。不是"数据要醒目"，是"4 个卡片，数字 64px"。

这是给实习生写 brief 的方式。**模糊指令换不来精准产出**。

### 让 AI 局部改的 8 条万能指令

这是第二级的核心技能。新手最常犯的错——每次不满意都让 AI 重写整个，结果**越改越乱**，因为它每次重写都会重新发挥，把上一版你已经改好的部分一起改坏。

正确姿势：**定位 + 改动 + 保留其他**。下面 8 条指令，直接抄：

1. "把第 3 页标题改成 XXX，**其他不动**"
2. "整体配色换成藏青 + 银白，**其他不动**"
3. "在第 5 页之后**插入一页**，内容是 XXX"
4. "把第 6 页的字体放大 20%，**只改第 6 页**"
5. "封面太花了，**只把封面**简化成只保留主题和年份"
6. "第 4 页的时间轴改成竖向，**只改这一页**"
7. "末页加上一句古诗词，要 14 字以内，**别动其他页**"
8. "第 2 页的数字 8.6 万改成 9.2 万，**别动其他文字**"

每条指令都有"其他不动"或"只改 X"——这四个字，是你跟 AI 协作的护城河。

### 这一级最常踩的两个坑

**坑 1：AI 编数字。**

AI 给你的"Q1 营收增长 23.7%"看起来很合理，但其中可能有 1 到 2 个数字是它编的。

它怎么编的？它见过几百万份 PPT，知道 Q1 营收增长的数字大概在 5%-30% 这个区间，于是它就在这个区间里随便挑一个看起来漂亮的，给你。

**对策**：每个数字都要从你自己的数据源核一遍。AI 给的提示词留着，但数字让它用 `[Q1 营收增长率]` 这样的占位，等 HTML 生成完，你自己把占位换成真数据。

**坑 2：AI 改的时候把好的部分一起改坏。**

你说"第 3 页标题改成 X"，它给你新代码，但你回看第 5 页，配色变了。

**对策**：每次改完，**翻一遍所有页**，对比上一版有没有意外变动。养成这个习惯，不然到投屏那一刻发现整个崩了。

### 红线（嵌在这一级）

**凡引用必查原文，凡数字必复核。**

AI 会幻觉。它可能给你编一段听起来很有道理、但根本不存在的"某某书里说"。这种引用如果你照样放进 PPT，演讲的时候被人问出处，你会很狼狈。

任何引用、名言、数据出处，都要去原文核对，**一字不差**。原文找不到的，宁可不用。

## 五、第三级 · 让人记住的 PPT

第二级是"不丢人"，第三级是"散会还有人主动来问怎么做的"。

差距在哪？三个东西：**视觉密度、节奏、收尾**。

### 在 Q1 复盘上再加一层：让它"惊艳"

把下面这段长提示词，发给 DeepSeek：

```
现在我们做最后一轮升级。请按以下规范重做整份 HTML：

视觉系统：
- 加纸纹背景纹理（用 SVG pattern 或 CSS 渐变实现）
- 每页右上角加章节序号水印（如"壹""贰""叁"），半透明
- 数据页加数字递增动画（数字从 0 涨到目标值，1.5 秒）
- 时间轴页用 SVG 画连接线和节点，节点可以 hover 高亮

交互：
- 顶部加一个细进度条，随翻页填充
- 右侧加圆点缩略导航，点击直接跳转
- 按 F 键全屏，按数字键跳转到对应页

收尾：
- 末页放一句 14 字以内的金句，使用大号衬线字体居中显示
- 金句下方一行小字"汇报人：[姓名]｜2026 Q1"

请保持现有的内容结构和配色不变，只增加上述视觉与交互元素。完整代码全发给我。
```

发完等大约 90 秒。这一版出来，效果会让你自己愣一下。

### 这一级真正的窍门：迭代不是修补

到这一级你会发现，最难的不是写提示词，是**判断什么时候该停**。

新手的本能是——"再让它加点动画"、"再加一页吧"、"封面再炫一点"——结果加到一定程度，整份 PPT 变成了**马戏团**。每页都在抢注意力，反而没有重点。

判断标准很简单：**如果这一版你已经能在领导面前讲完整 8 分钟而不脸红，就停。**

惊艳不是堆元素堆出来的，是节奏感。前面规规矩矩，关键一页突然来一下，反而让人记住。

### 红线（嵌在这一级）

**人名 / 单位名 / 职务，永远自己填，不让 AI 写。**

AI 不知道你公司的全称、领导的具体职务、内部称呼习惯。让它写，**100% 出错**。

提示词里直接写"姓名职务请用 `[XXX]` 占位，我自己填"。HTML 生成完，你用记事本打开，把所有 `[XXX]` 替换成真实信息。这一步快得不得了，但能救你的命。

## 六、投屏前一定要做的三件事

PPT 做出来再漂亮，投屏崩了等于零。这三件事不能省：

1. **提前 10 分钟到会议室**，把 U 盘插上，打开 .html 文件，按 F11 全屏，方向键试一遍翻页。能跑就关掉，等开始。
2. **会议室电脑大概率装的是 Edge**，不是 Chrome。Edge 完全能跑这种 HTML PPT，不用慌。
3. **U 盘里同时放一份 PDF 备份**——浏览器打开 .html，按 `Ctrl + P`（Mac 是 `Cmd + P`）选"另存为 PDF"。万一会议室电脑出问题，PDF 总能放。这一步的成本是 30 秒，性价比无敌。

## 七、这套方法的边界

老实说三遍——**它不适合**：

- **发给同事 / 领导用 WPS 改**。HTML 不能在 PowerPoint 里编辑，这条死活绕不开。
- **打印归档发文**。HTML 转 PDF 能打印，但格式不是体制内常见的"PPT 转 PDF"那种纸感，归档可能被打回。
- **上传到必须 .pptx 格式的内部系统**。系统就是不收，没办法。

如果你的 PPT 最终命运是上面这三种之一，老老实实用 WPS 套模板，AI 只用来润色文字。

但是它**特别适合迁移到**：

- **简历**（一个 .html 文件，HR 在浏览器里点开就能看，比 PDF 还鲜活）
- **读书会 / 分享会演示**（你自己讲，自己控场）
- **个人作品集**（投屏给面试官看）
- **节庆纪念 / 年终总结**（自己讲完发给参会人留念）
- **教学课件**（学生看一次就记得住）

把"Q1 项目复盘"这个案例换成上面任何一个主题，提示词结构完全一样。**这就是这套方法真正的价值——它不是 5 个模板，是一种使唤 AI 的方式**。
